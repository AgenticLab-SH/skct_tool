🧭 운영 가이드

헷갈리기 쉬운 핵심 구분
🔗 사이트 보기: 실제 운영 사이트를 개발자 우회로 여는 버튼입니다. 저장 후 운영 동작을 바로 확인할 때 사용합니다.
🧪 테스트 사이트: 숨겨진 스테이징 사본입니다. 운영 데이터는 읽기만 하고, 글/댓글/좋아요/방문기록 저장은 막혀 있습니다.
🚧 접속 차단: 일반 사용자를 점검 페이지로 보내는 설정입니다. 개발자는 `사이트 보기`로 계속 확인할 수 있습니다.
배포 전 체크
1. 먼저 🧪 테스트 사이트에서 UI와 흐름 확인
2. 저장 후 🔗 사이트 보기로 운영 동작 재확인
3. 점검 문구와 완료 일시가 맞는지 확인
4. 마지막에만 접속 차단 해제
자주 바꾸는 설정
- 타이머 기본값: 전체/과목/쉬는시간
- 가이드 초기값: 문항당 권장 시간, 표시 여부
- 높이 비율: 타이머, 메모/그림판, 계산기 영역
- 공지: Help 모달, 커뮤니티 상단 공지
문제 발생 시 우선 확인
- 운영은 사이트 보기에서 재현되는지
- 스테이징만 이상하면 테스트 사이트 문제인지
- 저장 후 값이 안 바뀌면 새로고침으로 재확인
- 점검 중인데 일반 접속이 되면 접속 차단 체크 상태 확인
기억할 점: 테스트 사이트는 운영 화면과 비슷하게 보는 용도이지, 운영 DB를 수정하는 공간이 아닙니다. 실제 반영 여부는 항상 저장 → 사이트 보기로 한 번 더 확인하는 편이 안전합니다.

이 표는 관리자 전용 고급모드 계정 관리 영역입니다. 여기서 추가/수정/삭제한 계정 정보는 Firebase `config/advancedFeatureConfig`에 저장되며, 공개 페이지에는 평문 비밀번호가 노출되지 않습니다. 직접 추가한 계정은 이메일이 없어도 로그인 ID + 비밀번호만 맞으면 바로 로그인됩니다.
계정 입력 기준 보기 처음 등록할 때만 확인하면 됩니다.
사용자 식별 이름, 이메일처럼 누구 계정인지 알아보기 위한 관리자용 표시입니다.
로그인 ID 로그인할 때 사용자가 직접 입력하는 계정 ID입니다. 이메일처럼 생긴 값도 쓸 수 있고, 관리자 수기 등록처럼 이메일이 없는 계정도 이 값 하나로 로그인됩니다.
비밀번호 관리자 화면에서는 평문으로 봅니다. 신청번호가 연결된 기존 계정은 보강 저장 버튼으로 평문 저장까지 한 번에 처리할 수 있습니다.
만료일 선택한 날짜 23:59까지 사용할 수 있습니다. 비워두면 영구이용권으로 저장됩니다.
